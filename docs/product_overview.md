# LocalTrip Builder ポートフォリオ説明文

## 短い説明

LocalTrip Builder は、旅行条件に合わせて地方観光プランを作成する Django 製Webアプリです。観光スポット管理、候補抽出、生成AI API連携、Google Maps URL生成、PDF出力までを一通り実装しています。

## 提案・応募時の説明文

LocalTrip Builder は、地方観光向けの観光プラン作成支援アプリです。

Django Admin で地域・観光カテゴリ・観光スポットを管理し、旅行者が入力した条件に合わせて候補スポットを抽出します。その候補情報をもとに、Mock / Gemini / OpenAI / Claude のいずれかを選択してAI観光プランを生成できます。

外部APIを扱う前提のため、APIキーは `.env` で管理し、画面表示やテスト実行だけでは外部APIを呼び出さない設計にしています。Google Mapsについても、通常画面ではAPIキー不要のURL生成を使い、Geocoding APIは管理者が明示的に実行した場合だけ呼び出す構成です。

生成結果はDBに保存し、PDFとしてダウンロードできます。ポートフォリオ用に、外部APIを呼ばずに動作確認できるMock生成とデモデータ投入コマンドも用意しています。

## 技術的な見どころ

- Django / PostgreSQL を使った業務アプリ構成
- `apps/` 配下に機能単位でアプリを分離
- 観光スポット、旅行条件、AI生成結果を分けたモデル設計
- AIプロバイダーを共通インターフェースで切り替える設計
- 外部APIをPOST操作時だけ呼び出す安全な導線
- Google Maps APIを呼ばないURL生成と、管理者操作に限定したGeocoding連携
- ReportLabによるPDFレスポンス生成
- `.env` / `.gitignore` / README / docs を含む公開前整備

## デモで見せるポイント

1. Django Admin で観光スポットを管理できること
2. 旅行条件から候補スポットが抽出されること
3. Mock生成で外部APIなしにデモできること
4. Gemini / OpenAI / Claude に差し替えられる設計であること
5. Google MapsリンクとルートURLをAPIキーなしで生成していること
6. 生成結果をPDFで出力できること

## 注意して説明する点

- デモデータはサンプルであり、実在情報の正確性を保証しません。
- 実APIを使う場合は、各APIキーと利用料金・利用規約の確認が必要です。
- 個人情報をAIプロンプトへ送らない運用設計が必要です。
- 本番公開時は `DEBUG=False`、HTTPS、許可ホスト、管理画面保護などの設定が必要です。
