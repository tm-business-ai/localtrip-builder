# LocalTrip Builder デモシナリオ

このドキュメントは、LocalTrip Builder をポートフォリオや商品デモとして見せるための手順です。

デモデータは宮崎県内の観光を題材にしたサンプルです。地域4件以上、カテゴリ9件以上、観光スポット12件以上を投入できます。営業時間、料金、定休日、緯度経度、説明文はデモ用サンプルであり、実在情報の正確性を保証するものではありません。

## 事前準備

依存関係をインストールします。

```bash
pip install -r requirements.txt
```

`.env.example` をコピーして `.env` を作成します。APIキーは未設定またはダミー値のままでも、Mock生成のデモは実行できます。

```bash
copy .env.example .env
```

PostgreSQLを起動し、マイグレーションを適用します。

```bash
docker compose up -d
python manage.py migrate
```

管理画面も見せる場合は、管理ユーザーを作成します。

```bash
python manage.py createsuperuser
```

## デモデータ投入

以下のコマンドで、地域・観光カテゴリ・観光スポットのデモデータを投入します。

```bash
python manage.py seed_demo_data
```

このコマンドは外部APIを呼び出しません。Google Maps API、Geocoding API、Gemini、OpenAI、Claudeへの通信も行いません。

データ投入は `slug` などの安定したキーを使って更新するため、複数回実行しても同じデモスポットが重複しにくい設計です。

## デモの流れ

開発サーバーを起動します。

```bash
python manage.py runserver
```

トップページを開きます。

```text
http://127.0.0.1:8000/
```

基本のデモ手順:

1. トップページでアプリ概要を説明します。
2. `http://127.0.0.1:8000/admin/` を開き、地域・観光カテゴリ・観光スポットが登録されていることを見せます。
3. `http://127.0.0.1:8000/plans/new/` を開き、旅行条件を入力します。
4. 地域に「宮崎市」「高千穂エリア」「日南海岸エリア」「霧島周辺エリア」のいずれかを選択します。
5. カテゴリ、旅行日数、移動手段、旅行者タイプ、雨の日想定、子連れ向け優先などを指定します。
6. プレビュー画面で候補スポット、Google Mapsリンク、Google MapsルートURL、AIプロンプト下書きを確認します。
7. AIプロバイダーは `Mock` を選択して生成します。
8. 生成結果画面でAI観光プラン、入力条件、候補スポット、使用プロンプトを確認します。
9. 「PDFをダウンロード」からPDF出力を確認します。

## 推奨デモ条件

### 宮崎市の日帰りデモ

- 地域: 宮崎市
- 旅行日数: 1日
- 移動手段: 車
- カテゴリ: 自然、グルメ、お土産
- 旅行者タイプ: 家族旅行
- 子連れ向けを優先: ON
- 補足条件: 海沿いを散策し、地元グルメと買い物も楽しみたい

### 高千穂エリアの自然・文化デモ

- 地域: 高千穂エリア
- 旅行日数: 1日
- 移動手段: 車
- カテゴリ: 自然、神社・仏閣、体験
- 旅行者タイプ: カップル
- 補足条件: 景色と神話ゆかりの場所をバランスよく回りたい

### 日南海岸エリアの雨の日向けデモ

- 地域: 日南海岸エリア
- 旅行日数: 1日
- 移動手段: 車
- カテゴリ: グルメ、温泉、歴史
- 旅行者タイプ: 友人同士
- 雨の日を想定: ON
- 補足条件: 雨でも立ち寄りやすい場所を優先したい

### 霧島周辺エリアのドライブデモ

- 地域: 霧島周辺エリア
- 旅行日数: 1日
- 移動手段: 車
- カテゴリ: 絶景、温泉、カフェ
- 旅行者タイプ: 友人同士
- 補足条件: 高原の景色を楽しみ、途中でカフェ休憩と温泉も入れたい

## デモ時の注意

- APIキーや `.env` の内容は画面共有しないでください。
- Gemini / OpenAI / Claude を選択すると、APIキー設定済みの場合は実APIを呼び出します。ポートフォリオデモでは基本的に `Mock` を使ってください。
- Google MapsリンクとルートURLはAPIキー不要のURL生成です。画面表示やPDF出力だけではGoogle Maps APIを呼び出しません。
- Geocoding APIはDjango Adminの明示的なアクションでのみ実行されます。通常のデモでは実行しないでください。
- 旅行者の氏名、電話番号、メールアドレス、住所、予約番号などの個人情報は入力しないでください。
